## 七、关键特性说明

### 7.1 混合检索策略

| 检索方式 | 技术实现 | 优势 | Top-K |
|---------|---------|------|-------|
| 向量检索 | ChromaDB + BGE-Large-zh | 语义理解、模糊匹配 | 20 |
| 关键词检索 | BM25 + Jieba | 精确匹配、专有名词 | 20 |
| 图谱检索 | Neo4j Cypher | 逻辑推理、关系查询 | 10 |

### 7.2 四智能体协作机制

```
激进派 (Radical)
  ↓ 提取尽可能多的知识
保守派 (Conservative)
  ↓ 过滤低置信度结果
对抗派 (Adversarial)
  ↓ 检测冲突和矛盾
大法官 (Judge)
  ↓ 最终裁决和融合
```

### 7.3 查询改写机制

- **指代消解**：将"它"、"这个"等代词替换为具体实体名
- **语义补全**：将不完整问题补全为独立检索句
- **智能审计**：34 项指代词审计，防止检索漂移

### 7.4 重排序机制

使用 BGE-Reranker-Large (Cross-Encoder) 对混合检索结果进行重新打分：

```
Input: Query + Candidate Context
Output: Relevance Score (0-1)
```

比传统的 Bi-Encoder 更精确，但计算成本更高。

## 九、性能指标

| 指标 | 数值 | 说明 |
|-----|------|------|
| 查询改写延迟 | ~1-2s | DeepSeek API 调用 |
| 向量检索延迟 | ~100-200ms | ChromaDB 本地查询 |
| BM25 检索延迟 | ~50-100ms | 内存索引 |
| 图谱检索延迟 | ~200-500ms | Neo4j Cypher 查询 |

| **总延迟** | **~5-10s** | 端到端响应时间 |

## 十、扩展性设计

### 10.1 水平扩展

- **向量数据库**：ChromaDB 可替换为 Milvus/Qdrant（支持分布式）
- **图数据库**：Neo4j 可升级为企业版（支持集群）
- **LLM 服务**：可接入多个 API 端点实现负载均衡

### 10.2 垂直扩展

- **GPU 加速**：本地部署 BGE 模型可利用多卡并行
- **缓存机制**：可增加 Redis 缓存热门查询结果
- **异步处理**：已使用 asyncio + nest_asyncio 实现异步调用

### 10.3 功能扩展

- **意图识别**：可在 Query Rewriter 前增加意图分类模块
- **多模态支持**：可扩展图片、表格的理解能力
- **实时更新**：可增加增量索引更新机制
