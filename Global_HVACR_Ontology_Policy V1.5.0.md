## 1. 统一语义底座 (13x13 矩阵)

### 1.1 实体类型定义 (Entity Types)
1.  **System**: 完整功能的系统集合。示例：净化空调系统、自控系统。
2.  **Space**: 物理或逻辑空间。示例：洁净室、防火分区、技术竖井。
3.  **Equipment**: 独立运行的主机。示例：风机盘管、冷水机组、高效过滤器。
4.  **Component**: 管路附件或末端。示例：风管、防火阀、止回阀、静压箱。
5.  **Parameter**: 物理属性或技术指标。示例：温度、耐火极限、压差、风速。
6.  **Value**: 标准化数值。**格式强制：[运算符][数值][Unicode单位]**。示例：≥1.0h、≤0.5μm。
7.  **Material**: 建筑或工程材料。示例：不燃材料、彩钢板、岩棉、橡塑海绵。
8.  **Standard**: 外部引用标准。示例：GB 50073-2013、ISO 14644。
9.  **Reference**: 文档内部章节或表号。示例：条款6.2.3、表3.0.1。
10. **Action**: 名词化措施/功能。示例：温湿度监测、压差控制、变频调节。
11. **Method**: 计算/检测方法或公式。示例：缝隙法、公式3.0.1。
12. **Concept**: 术语定义或理论状态。示例：单向流、空态、静态。
13. **Note**: 条文说明或补充引用。示例：条文说明P79、附录B。

### 1.2 关系类型定义 (Relation Types)
1.  **IS_A**: 分类/等价。
2.  **PART_OF**: 空间/系统组成。
3.  **LOCATED_IN**: 物理部署位置。
4.  **HAS_PARAMETER**: 实体/材料关联技术参数。
5.  **HAS_VALUE**: 参数赋值（含逻辑运算符）。
6.  **CONTROLS**: 动作/设备对参数的调节。
7.  **REQUIRES**: 强制性要求（Positive/Negative）。
8.  **COMPLIES_WITH**: 符合标准。
9.  **DEFINED_IN**: 物理来源。
10. **USES_METHOD**: 采用的方法/公式。
11. **CONNECTS_TO**: 拓扑连接/穿越。
12. **USES**: 非强制性配置（宜/可）。
13. **REFER_TO**: 指向注释或正文关联。

## 2. 行业防护墙 (Domain Guardrails)

### 2.1 术语保护清单 (禁止拆分)
以下术语必须作为整体实体处理，**严禁拆分**：
- 风机盘管 (Equipment)
- 高效过滤器 / 亚高效过滤器 / 超高效过滤器 (Equipment)
- 防火阀 / 调节阀 / 止回阀 / 密闭阀 / 余压阀 / 排烟阀 (Component)
- 静压箱 (Component)
- 空气吹淋室 (Space)
- 传递窗 / 真空吸尘器 (Equipment)

### 2.2 复合实体处理与耦合逻辑
1. **参数剥离规则**：若复合词末尾为参数（如"风管风速"），必须拆分为 `Component + Parameter`。
2. **材料-性能强耦合规则**：当原文出现"具有[特定性能]的[材料]"时，必须建立 `Material` --`HAS_PARAMETER`--> `Parameter` --`HAS_VALUE`--> `Value` 完整链条。

**决策树处理规则**：
1. **是否描述材料性能？**
   - 是 → 进入步骤2
   - 否 → 按普通实体处理
   
2. **性能描述类型？**
   - **单一性能**（如"耐火极限不低于1.0h的不燃材料"）→ 建立完整链条
   - **多个性能**（如"耐火极限≥1.0h且导热系数≤0.045W/(m·K)的岩棉"）→ 建立多条并行链条
   - **性能范围**（如"耐温范围-20℃~120℃的橡塑海绵"）→ Parameter为"耐温范围"，Value为"-20~120℃"
   - **性能分类**（如"A级防火材料"）→ Parameter为"防火等级"，Value为"A级"
   - **负向约束**（如"不得使用可燃材料"）→ 只建立`REQUIRES`关系

### 2.3 关系触发词与Unicode转码标准
| 物理量/符号 | 原始输入示例 | **强制转换目标(Unicode)** |
|-------------|--------------|---------------------------|
| 立方米/小时 | m3/h, m^3/h  | **m³/h**                  |
| 平方米      | m2, m^2      | **m²**                    |
| 负一次方    | h-1, /h      | **h⁻¹**                   |
| 微米        | um, micron   | **μm**                    |
| 微克        | ug           | **μg**                    |
| 摄氏度      | degC, C      | **℃**                     |

## 3. 单位推断与物理常识熔断协议

### 3.1 分级预警机制 (Alarming Levels)
大法官在处理推断数值时，需遵循以下分级：
- **Normal (正常)**: 置信度按 4.3 节查表判定。
- **Warning (异常预警)**: 数值偏离常规但逻辑可能成立。**置信度强制降至 0.3**。
- **Critical (幻觉熔断)**: 数值物理上不可能。**置信度设为 0.1，直接剔除**。

### 3.2 物理极值分级表
| 参数类别 | 正常范围 (Normal) | 预警阈值 (Warning) | 熔断阈值 (Critical) |
|----------|-------------------|-------------------|-------------------|
| **压差** | 0 ~ 100Pa | 100 ~ 500Pa | < 0 或 > 500Pa |
| **洁净室温度** | 15 ~ 30℃ | 5 ~ 15℃ 或 30 ~ 50℃ | < 0 或 > 100℃ |
| **风速** | 0.2 ~ 10m/s | 10 ~ 30m/s | < 0 或 > 100m/s |
| **换气次数** | 10 ~ 60h⁻¹ | 60 ~ 200h⁻¹ | < 0 或 > 1000h⁻¹ |
| **耐火极限** | 0.5 ~ 3.0h | 3.0 ~ 4.0h | < 0 或 > 8.0h |
| **湿度** | 10% ~ 90% | 0% ~ 10% 或 90% ~ 100% | < 0% 或 > 100% |
| **噪声级** | 30 ~ 100dB(A) | 20 ~ 30dB(A) 或 100 ~ 120dB(A) | < 20dB(A) 或 > 120dB(A) |

## 4. 冲突解决与仲裁矩阵

### 4.1 优先级决策链
1. **Critical极值熔断 > 全局规则**: 任何触发熔断的数据不予进入图谱。
2. **术语保护 > 复合拆分**: 涉及2.1清单时，强制撤销拆分。
3. **Unicode标准化 > 原始描述**: 最终Value必须使用Unicode字符。

### 4.2 策略分歧裁决
以下情况为策略分歧，对抗派**严禁报错**，大法官按规则处理：
- **Note关联方向**: 激进派建立 `Note` --`REFER_TO`--> `Reference`。
- **正负向约束标记**: 激进派在 `description` 中标注 `[负向约束]`。
- **宜/可语句**: 保守派忽略，激进派提取为 `USES`。

### 4.3 置信度终审判定标准（优先级覆盖制）
大法官应依据以下优先级，直接赋予最终置信度，**不再进行累加计算**：

1.  **[Score=1.0] 铁证**：原文直接表述（含单位/运算符），且两派提取无实质冲突。
2.  **[Score=0.9] 继承**：数值无单位，但通过“表格继承”或“Note修正”补全了单位/数值。
3.  **[Score=0.8] 推断**：激进派基于 `{previous_context}` 上下文推断单位，且符合物理常识；或拆分了非保护术语的复合实体。
4.  **[Score=0.6] 映射**：仅有激进派基于“默认单位映射表”补全单位；或从“宜/可”语句中提取的 `USES` 关系。
5.  **[Score=0.3] 存疑**：触发 **Warning** 级物理异常预警；或激进派无明确依据的推断。
6.  **[Score=0.1] 剔除**：触发 **Critical** 级熔断；严重本体违规；事实幻觉。

## 5. 结构化解析流程规范

### 5.1 表格解析扩展
- **跨块表头继承**: 必须从 `{previous_context}` 继承表头单位。
- **Note关联**: 若表格下方有注释，必须建立 `Note` --`EXPLAINS`--> `表[编号]` 关系。

### 5.2 运算符转换映射
- "不小于"、"及以上"、"以上"、"不低于" → **≥**
- "不大于"、"及以下"、"以下"、"不超过" → **≤**
- "范围为"、"介于...之间" → **[下限]~[上限]**（如 20~22℃）

## 6. Unicode标准化执行细则

### 6.1 希腊字母与特殊符号映射表
| 名称 | ASCII/错误形式 | **Unicode正确形式** | Unicode编码 |
|------|---------------|-------------------|-------------|
| 微米 | um, mu | **μm** | U+03BC |
| 阿尔法 | alpha | **α** | U+03B1 |
| 贝塔 | beta | **β** | U+03B2 |
| 伽马 | gamma | **γ** | U+03B3 |
| 欧米伽 | omega | **ω** | U+03C9 |
| 角度 | degree | **°** | U+00B0 |
| 平方 | ^2 | **²** | U+00B2 |
| 立方 | ^3 | **³** | U+00B3 |
| 负一次方 | ^-1 | **⁻¹** | U+207B U+00B9 |

### 6.2 复合单位拼接规则
1. **先上标后其他**：先处理上标（如³），再处理其他符号。示例："m3/h" → "m³/h"。
2. **分子分母处理**：使用"/"或"·"连接。示例："W/(m·K)" → 保持原格式。
3. **范围连接符**：统一使用"~"（U+007E）。
